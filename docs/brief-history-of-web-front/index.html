웹 프런트 엔드의 간략한 역사

#작성 중#front-end#CSR#SPA#SSR#JAM Stack
목차
  1. Static File 정적 파일을 서빙하기 - 나모 웹 에디터
    1. 웹 브라우저 주소 창에 URL을 치면 일어나는 일
    2. 정적 파일(static file) HTML, CSS, JS, 이미지
    3. url과 폴더 기반 라우팅
  2. SSR 서버에서 템플릿 동적으로 렌더링하기 - 템플릿 엔진
    1. 실시간으로 변하는 상태(state)에 따라 다른 내용을 보여주고 싶다.
    2. 동적 (dynamic) 렌더링이란?
  3. CSR 클라이언트에서 렌더링하기 - JQuery와 Ajax, 모던 JS
    1. 사용자 동작에 따라 반응하게 만들고 싶다.
  4. SPA 웹 앱을 만들자 - Angular, Vue, React
    1. 새로고침 없이 브라우저에서 동적으로 dom을 그리고 싶다.
  5. SSR이 SPA와 함께 부활하다 - Next.js
    1. 빠르게 사이트를 보여주고, 멍청한 검색엔진 크롤러에게 HTML을 보여주고 싶다
  6. 정적 파일이 현대 기술과 함께 돌아오다 - JAM
    1. 복잡한 WebServer 없이 빠르게 웹 사이트를 보여주고 싶다.
  7. 진화는 일직선이 아니다.
  8. 각 방식의 장, 단점 총 정리
